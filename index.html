Enlarge Interaction — 6 Options

Six approaches for the enlarge button on SAP.com product screenshots and videos. A–C open a separate view; D–F zoom right on the image itself.

Option A

Classic Lightbox

The screen dims, the image takes center stage at the largest possible size. Arrow keys and dots navigate. Exactly what users expect from an "enlarge" button — no surprises.

✓ Zero learning curve ✓ Max image real estate ✓ Works for images and video Best for: screenshots, demo videos

Option B

Lightbox + Info Panel

Same familiar dark overlay, but the image shares the screen with a white info panel on the right. The enlarged view doubles as a browsable gallery with full title, description, and thumbnail navigation.

✓ Familiar lightbox pattern ✓ Rich context alongside image ✓ Thumb strip for browsing ✗ Image slightly smaller than Option A Best for: feature tours, product galleries

Option C

Slide-up Sheet

A sheet rises from the bottom of the screen with the image at large size and a thumbnail strip below. The animation and drag-handle affordance feel natural on both desktop and mobile — familiar from iOS, Google Photos, and modern web apps.

✓ Mobile-native feel ✓ Intuitive upward motion ✓ Thumbnail strip stays visible ✗ Less common on desktop-first sites Best for: mobile-primary products

Option D

Magnifier Lens

Hovering the image reveals a circular magnifying-glass lens that follows the cursor and shows a 2.5× zoomed portion. Clicking still opens the full-screen lightbox. Zero clicks needed to start exploring — just move the mouse.

✓ Zero-click exploration ✓ Familiar from e-commerce (Amazon, Zalando) ✓ Image stays in context ✗ Doesn't work on touch Best for: detail-rich UI screenshots

Option E

Click to Zoom + Pan

Pressing the enlarge button zooms the image 2.5× right in place. Moving the mouse then pans around the zoomed image. Click again (or press Esc) to zoom back out. No overlay, no context switch — the image just gets bigger where it sits.

✓ Stays fully in page context ✓ Satisfying, direct interaction ✓ No overlay / focus trap ✗ Loses surrounding page while zoomed Best for: single featured screenshots

Option F

Picture-in-Picture Zoom

A floating zoom window sits in the corner of the image. As you move the mouse, the PiP window shows a 2.5× magnified crop of wherever the cursor is — with a reticle on the main image showing the exact region. Familiar from medical imaging, mapping, and high-end e-commerce.

✓ Main image always fully visible ✓ No overlay or context loss ✓ Precise — cursor position = zoomed area ✗ PiP window covers image corner Best for: dense UI screenshots with small labels
